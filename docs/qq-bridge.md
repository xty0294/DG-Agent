# QQ 桥接使用指南

通过 QQ 桥接功能，你可以在手机 QQ 上远程控制 DG-Agent，无需一直盯着电脑浏览器。

## 前置条件

- 一台运行 DG-Agent 网页的电脑（Chrome / Edge）
- 电脑上安装并运行 [NapCat](https://github.com/NapNeko/NapCatQQ)
- 一个用于做机器人的 QQ 账号（建议用小号）

## 第一步：安装并启动 NapCat

NapCat 是一个 QQ 机器人框架，提供 OneBot v11 WebSocket 接口。

### Windows

1. 前往 [NapCat Releases](https://github.com/NapNeko/NapCatQQ/releases) 下载`NapCat.Shell.Windows.OneKey.zip`
2. 解压后运行 `NapCatInstaller.exe` 安装必要文件(这步可能需要比较长的时间)，直到看见
   `press any key to continue`。
3. 进入 `NapCat.44498.Shell` 使用 `napcat.bat`打开
4. 启动后会弹出二维码，用**机器人的 QQ 号**在手机上扫码登录

### Linux / macOS

参考 NapCat 官方文档使用 Docker 或直接运行。

## 第二步：配置 NapCat

扫码登录成功后，NapCat 会在控制台输出 WebUI 的访问地址，通常为：

```
http://localhost:6099
```

用浏览器打开该地址，使用 NapCat 启动时显示的 Token 登录 WebUI。进入「网络配置」页面，添加一个**正向 WebSocket** 连接，配置如下：

| 配置项         | 建议值   | 说明                       |
| -------------- | -------- | -------------------------- |
| WebSocket 服务 | 启用     | 开启正向 WebSocket         |
| WS 监听端口    | `3001`   | DG-Agent 默认连接此端口    |
| 消息格式       | `string` | 使用字符串格式             |
| 上报自身消息   | 关闭     | 避免机器人回复自己触发循环 |
| Token          | 留空     | 除非你需要额外安全验证     |
| 心跳           | 默认即可 |                            |

## 第三步：在 DG-Agent 中配置

1. 打开 DG-Agent 网页，点击右上角 **⚙️ 设置**
2. 切换到 **桥接** 标签页
3. 开启 **启用社交平台桥接**
4. 在 **QQ** 子标签中：
   - 开启 **启用 QQ**
   - **NapCat WebSocket 地址**：`ws://localhost:3001`（默认值，通常不用改）
   - **允许的 QQ 用户号**：填入你**自己手机 QQ 的号码**（用于控制设备的那个号），多个用逗号分隔
   - **允许的 QQ 群号**（可选）：如果要在群里使用，填入群号
   - **权限模式**：
     - `每次确认` — 每次 AI 要操作设备时，会在 QQ 里发消息让你确认
     - `总是允许` — AI 直接操作，不需要确认
5. 点击 **保存并重新连接**

连接成功后，主页面会显示「桥接模式已开启」的覆盖层，并实时显示消息日志。

## 第四步：开始使用

### 私聊

直接给机器人 QQ 号发消息即可：

```
你：轻轻试一下 A 通道
机器人：好的，我先用很轻的力度让你感受一下~
```

### 群聊

在群里需要 **@机器人** 才会触发回复：

```
你：@机器人 把 A 通道强度调到 10
机器人：好的，已将 A 通道强度调整为 10~
```

## 权限确认

如果权限模式设为「每次确认」，AI 操作设备前会在 QQ 里发送确认消息：

```
机器人：
  AI 请求操作设备
  工具: start
  说明: 启动 A 通道：播放「呼吸」波形，强度 15

  回复数字选择：
  1. 允许本次
  2. 5分钟内都允许
  3. 总是允许（本会话）
  4. 拒绝

你：1
```

30 秒内未回复会自动拒绝。

## 常见问题

### 连接不上 NapCat

- 确认 NapCat 已启动且 WebSocket 服务已开启
- 确认端口号一致（默认 `3001`）
- 检查浏览器控制台（F12）是否有 `[QQ] Connected` 日志

### 机器人不回复

- 检查「允许的 QQ 用户号」是否填写了你的 QQ 号
- 群聊时检查是否 @ 了机器人
- 检查「允许的 QQ 群号」是否包含当前群
- 确认 NapCat 中「上报自身消息」为关闭状态

### 设备操作没有执行

- 确认浏览器端已通过蓝牙连接了郊狼设备
- 检查权限模式设置 — 如果是「每次确认」，需要在 QQ 回复数字确认
- 浏览器标签页不能关闭，需要保持打开状态

### 断线重连

QQ 适配器内置自动重连机制（3 秒起步，最长 30 秒间隔）。NapCat 重启后会自动恢复连接，无需手动操作。
